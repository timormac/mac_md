#### Doris简介

```mysql
#简介
Apache Doris 是一个快速的、基于列存储的分布式分析数据库，适用于OLAP（在线分析处理）场景。它最初由百度开发，目的是为了快速分析大规模数据集。 对标的是clickhouse和hive管理的输出仓库进行数据分析，数据获取。
doris数据分析比hive快，面向的是快速查询和实时分析设计的
doris和clickhouse都是mpp分析型数据库

#Doris 的特点包括：
列式存储： 优化了数据仓库的分析查询速度。
实时写入： 支持实时批量导入数据。
MPP架构： Doris 使用大规模并行处理（MPP）架构，能够快速执行查询操作。
易于扩展： 可以水平扩展以处理更多数据。
使用 Flink 和 Doris 的区别
#用途差异：
Flink 主要用于流式数据处理和复杂事件处理，非常适合实时分析和即时决策支持。
Doris 主要用于快速的数据分析和OLAP，适合数据仓库的实时分析查询。
#数据处理模式：
Flink 是一个全面的流处理引擎，同时支持实时流处理和批处理。
Doris 是一个OLAP数据库，更适合作为数据仓库，对实时或近实时数据进行分析。
#系统架构：
Flink 是一个计算引擎，不存储数据，通常需要与其他存储系统（如 Apache Kafka、HDFS）配合使用。
Doris 既是计算引擎也是存储系统，拥有自己的存储管理。
#性能特点：
Flink 专注于低延迟的流处理。
Doris 专注于快速的批量数据查询。
易用性：
Flink 需要用户编写程序来定义数据流转换和处理逻辑。
Doris 通常通过SQL接口提供服务，易于与传统的BI工具集成。
维护和管理：
Flink 需要较多的维护工作，包括管理作业的生命周期、监控和调优。
Doris 的维护相对简单，主要是集群和数据管理。
总结来说，如果你的需求是实时流处理、事件驱动的应用，Flink 是一个更合适的选择。而如果你的需求是快速的数据分析和报告，特别是需要使用SQL进行数据查询，那么Doris可能是一个更好的选择。在很多实际应用场景中，Flink 用于实时计算和数据处理，而计算结果存储到 Doris 中，利用 Doris 进行高效的数据分析和查询。
```
